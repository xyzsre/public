Cisco IOS Useful Command Library

Two hundred configuration, verification, and troubleshooting commands curated for Cisco routers and switches within GNS3. Organized by core disciplines so you can deploy, validate, and resolve lab scenarios quickly.

1. Device Access & System Management

Access

1. enable

Elevate to privileged EXEC mode. Example: type enable and enter your secret to manage configs.

Access

2. disable

Return to user EXEC mode. Useful after testing with limited permissions.

Access

3. configure terminal

Enter global configuration mode to make device-wide changes.

Access

4. exit

Leave the current configuration context. Example: exit interface mode back to global config.

System

5. write memory

Save the running configuration to startup-config. Essential before stopping labs.

System

6. copy running-config startup-config

Alternate syntax to persist configuration changes.

System

7. show running-config

Display the current running configuration for review or documentation.

System

8. show startup-config

View the configuration stored in NVRAM. Compare with current running-config.

System

9. erase startup-config

Clear saved configuration, often followed by reload for fresh lab state.

System

10. reload

Reboot the device. Use reload in 5 to schedule if you need rollback safety.

System

11. show version

Display IOS version, platform, uptime, and licensing details.

System

12. show inventory

List hardware modules and serial numbers (supported on certain platforms).

System

13. show environment all

Verify system temperatures, power supplies, and fans—helpful in hardware labs.

Access

14. terminal length 0

Disable pagination for current session—ideal when copying configs.

Access

15. terminal monitor

Enable logging messages to current terminal session, essential for remote console.

Access

16. logging synchronous

Prevents syslog messages from interrupting CLI input on console lines.

System

17. service timestamps debug datetime msec

Add precise timestamps to debug output for correlation.

System

18. service timestamps log datetime msec

Apply timestamps to syslog messages for accurate event tracking.

Access

19. show users

Display users currently logged in. Useful for multi-user labs.

Access

20. clear line vty

Disconnect stuck or unauthorized VTY sessions.
